🏗️ 系统架构

Direct 模式（内网部署）

┌─────────────────┐      HTTP       ┌─────────────────┐     fly-pigeon    ┌─────────────────┐
│   MCP Server    │ ───────────────▶│   HIL Server    │ ─────────────────▶│   企业微信       │
│  (AI Agent)     │ ◀───────────────│   (内网部署)     │ ◀─────────────────│   (飞鸽传书)     │
└─────────────────┘                 └─────────────────┘      回调          └─────────────────┘

✓ 适用场景：

  • MCP Server 和 HIL Server 都在内网运行
  • 可以直接访问飞鸽传书 API
  • 简化部署，无需额外的 Worker 组件
  • 响应速度快，架构简单

🛠️ MCP 工具

📤

send_and_wait_reply

发送消息到企业微信并等待用户回复。

参数:
  message: 要发送的消息内容
  chat_id: 目标群或私聊 ID (可选)
  image_paths: 图片路径列表 (可选)
  project_name: 项目名称 (可选)

返回:
  status: success | timeout | error
  replies: 用户回复列表
  message: 描述信息
📨

send_message_only

仅发送消息，不等待回复。适用于通知场景。

参数:
  message: 要发送的消息内容
  chat_id: 目标群或私聊 ID (可选)
  image_paths: 图片路径列表 (可选)
  project_name: 项目名称 (可选)

返回:
  status: success | error
  message: 描述信息
