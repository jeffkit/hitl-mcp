🚀 1. 快速开始（2 分钟）

📱 第一步：添加企微机器人

在企业微信中搜索并添加 HITL-Walle 机器人
向它发送任意消息，机器人会回复详细的接入指南和你的 Chat ID

⚙️ 第二步：配置 Agent 的 MCP

以 Cursor 为例，在 MCP 配置文件（~/.cursor/mcp.json）中添加：

~/.cursor/mcp.json 
{
  "mcpServers": {
    "wecom-hil": {
      "command": "uvx",
      "args": [
        "hitl-mcp",
        "--service-url", "https://hitl.woa.com/api",
        "--chat-id", "你的Chat ID"
      ]
    }
  }
}

⚠️ 注意事项

  • 将上面的 你的Chat ID 替换为机器人回复给你的实际 Chat ID
  • Windows 用户请将 uvx 改为 uvx.exe
  • 其他 Agent（如 Claude Desktop、Windsurf 等）请参考各自的 MCP 配置文档

🎯 第三步：配置 Feedback Protocol（强烈推荐）

在 Cursor 的 Rules 或 .cursorrules 文件中添加以下规则，让 AI 学会主动汇报工作：

.cursorrules 或 Cursor Rules 
**Feedback Protocol**  

**Core Rule:** Always use the `send_and_wait_reply` tool to verify your work - never assume you got it right.  

### **When to Request Feedback:**  
1. **When You Need Clarification**  
   - If any part of the request is unclear or missing details  
   - Example: *"Should I focus more on [A] or [B] for this analysis?"*  

2. **When You Complete a Task**  
   - After delivering any result, big or small  
   - Example: *"Here's the draft: [summary]. Does this meet your needs?"*  

**How to Use the Tool Properly:**  
- Always phrase requests clearly and specifically  
- Offer options when possible (e.g., "Should I use approach X or Y?")  
- Never proceed without getting explicit confirmation  

**Golden Rule:** Your work isn't done until the user says "This is exactly what I needed." Keep improving until you hear those words.

🎉 完成！

配置完成后，AI 遇到疑问或完成任务时，会自动发消息到你的企业微信征求意见。你可以：

  • 📱 在手机上随时随地审查 AI 的工作
  • 💬 直接回复你的意见，AI 会根据反馈继续改进
  • 🔄 多轮迭代，直到满意为止

💡 提示

配置生效后，你会在 Agent 的工具列表中看到 send_and_wait_reply 和 send_message_only 两个工具。

🚀 2. 使用方法

工具 1: send_and_wait_reply

发送消息到企业微信并等待用户回复。这是最常用的工具。

使用示例 
工具参数:
  message: "请确认是否继续执行此操作？"
  project_name: "my-project"  (可选)
  image_paths: ["/path/to/screenshot.png"]  (可选)

返回结果:
  {
    "status": "success",
    "replies": [
      {
        "msg_type": "text",
        "content": "确认，可以继续",
        "from_user": {"name": "张三", "alias": "zhangsan"},
        "timestamp": "2024-01-01T10:30:00"
      }
    ],
    "message": "收到 1 条回复"
  }

工具 2: send_message_only

仅发送消息，不等待回复。适用于通知场景。

使用示例 
工具参数:
  message: "任务已完成！"
  project_name: "my-project"  (可选)

返回结果:
  {
    "status": "success",
    "message": "消息发送成功"
  }

多会话场景

当多个项目同时发送消息时，系统会在消息前添加 [#short_id project_name] 标识。

用户回复方式：

  • 单会话场景：直接回复即可
  • 多会话场景：使用「引用回复」功能精确选择要回复的消息

💡 引用回复

在企业微信中长按消息，选择「引用」，然后输入回复内容。系统会自动匹配到正确的会话。

💡 3. 最佳实践

1. 为不同项目使用不同的 Project Name

通过 --project-name 参数区分不同项目，方便在多项目并行时识别消息来源。

"args": [
  "hitl-mcp",
  "--service-url", "https://hitl.woa.com/api",
  "--chat-id", "your-chat-id",
  "--project-name", "frontend-project"  // 前端项目
]

2. 合理设置超时时间

根据任务特点设置合适的超时时间：

  • 快速确认：300 秒（5 分钟）
  • 一般任务：1200 秒（20 分钟，默认）
  • 复杂审批：3600 秒（1 小时）

3. 使用引用回复避免混淆

在多个会话同时进行时，建议使用企业微信的「引用回复」功能，确保回复准确传达。

4. 配合 Feedback Protocol 使用

强烈推荐配置 Feedback Protocol 规则，让 AI 主动寻求反馈， 避免方向性错误，提高工作效率。

5. 利用图片功能

对于 UI 相关的任务，可以让 AI 发送截图到企业微信，方便直观地确认效果。

🔧 4. 常见问题

Q: 出现 502 Bad Gateway 错误

原因：通常是设置了 HTTP 代理。

解决方案：在 MCP 配置中禁用代理：

"env": {
  "http_proxy": "",
  "https_proxy": "",
  "all_proxy": ""
}

Q: 机器人没有回复

可能原因：

  1. Chat ID 配置错误
  2. 机器人未加入该群
  3. 服务未正常运行

解决方案：

  1. 检查 Chat ID 是否正确（重新 @机器人 获取）
  2. 确认机器人在群中
  3. 访问 /health 检查服务状态

Q: 如何获取私聊的 Chat ID？

直接私聊机器人发送任意消息，机器人会自动回复 Chat ID。

Q: 多个项目同时发消息怎么区分？

使用「引用回复」功能。系统会在每条消息前添加 [#short_id project_name] 标识，用户引用回复时会自动匹配。

Q: 超时后会怎样？

会话会自动清理，AI 会收到超时状态。如果用户之后再回复，机器人会提示该会话已过期。

Q: 如何配置不同群组的提示消息？

访问 管理台，在"空闲提示配置"标签页中为不同的 Chat ID 配置自定义消息。
