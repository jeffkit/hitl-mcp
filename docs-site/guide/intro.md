# 什么是 hitl-mcp

`hitl-mcp` 是一个 **MCP 服务**，让 AI Agent（Cursor、Claude Desktop、Claude Code 等）能够在执行关键操作之前，通过 **微信** 或 **企业微信** 给你发一条消息，并 **等你回复** 再继续。这就是所谓的 Human-in-the-Loop（人在回路）。

举几个真实的例子：

- AI 准备执行一条 `git push --force`，先发微信问你：「确认强推到 main 吗？」——你回复 `OK` 它才执行。
- AI 跑完一个长任务，把结果总结成一条消息发到你的企微群，你不用一直盯着终端。
- AI 在删除一批文件前，把清单发给你确认，避免误删。

## 它是怎么工作的

```
┌──────────┐   MCP 协议   ┌─────────────┐    HTTP    ┌──────────────┐    微信/企微
│ AI Agent │ ───────────▶ │  hitl-mcp   │ ─────────▶ │  hitl-server │ ─────────▶  📱 你
│ (Cursor) │ ◀─────────── │  (MCP 端)   │ ◀───────── │  (本地后端)  │ ◀────────
└──────────┘  等待回复结果 └─────────────┘  轮询回复  └──────────────┘   你的回复
```

1. **AI Agent** 通过 MCP 协议调用 `send_and_wait_reply` 工具。
2. **hitl-mcp**（MCP 客户端）把消息转发给本地的 `hitl-server`。
3. **hitl-server** 通过某个引擎（微信 / 企微）把消息送到你手机。
4. 你在微信 / 企微里回复。
5. `hitl-server` 拿到回复，`hitl-mcp` 轮询取回，交给 AI。
6. AI 拿到你的回复，继续往下执行。

## 两个引擎怎么选

hitl-mcp 目前支持两个「引擎」，也就是两条不同的消息通道：

| 引擎 | 通道 | 适合谁 | 需要准备什么 |
|------|------|--------|------------|
| **ilink**（微信 ClawBot） | 个人微信 | 个人开发者 | 微信扫码登录一次 |
| **wecom-aibot**（企微 AI 机器人） | 企业微信 | 团队 / 企业用户 | 企微后台的 Bot ID + Bot Secret |

::: tip 个人用户
如果你只是想让 AI 在自己手机上问你，选 **ilink**，扫码就能用，不用申请任何企业账号。
:::

::: tip 企业团队
如果你希望 AI 在企微群里和团队互动，选 **wecom-aibot**，需要企微管理员开通 AI 机器人。
:::

## 你需要准备什么

不管选哪个引擎，整体只需要两样东西：

1. **一个 MCP 客户端**：Cursor、Claude Desktop、Claude Code 等支持 MCP 的工具。
2. **一台本地机器**（macOS / Linux）跑 `hitl-server`：它是消息的中转后端，跑在你本机即可，不需要公网 IP。

> `hitl-server` 默认只监听 `127.0.0.1:8081`，仅本机访问，不会暴露到公网。
